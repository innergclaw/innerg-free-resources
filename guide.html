Beginner Guide: Set Up Telegram + Voice Replies in OpenClaw

If you are brand new to OpenClaw, this guide walks you through a simple beginner setup so your assistant can respond in Telegram and speak with voice enabled.

Beginner friendly Telegram bot setup Voice-ready workflow

What you need first

  • An OpenClaw install already running on your machine
  • A Telegram account
  • Access to @BotFather inside Telegram
  • Your OpenClaw workspace and config available locally

Step-by-step

Step 1

Create your Telegram bot

Open Telegram, message @BotFather, and run /newbot. Follow the prompts and save the bot token somewhere safe.

Step 2

Enable Telegram in OpenClaw

In your OpenClaw config, turn Telegram on and add your bot token. Use pairing mode for DMs if you want controlled access.

Beginner default: set Telegram enabled, add your botToken, and use dmPolicy: "pairing".
Step 3

Restart OpenClaw so Telegram loads

After saving the config, restart the gateway so the Telegram channel comes online.

Step 4

Start a DM with your bot

Send your bot a message from your Telegram account. With pairing enabled, OpenClaw will wait for you to approve that DM before it becomes active.

Step 5

Approve the pairing request

Approve the Telegram pairing code inside OpenClaw. Once approved, your Telegram account can talk directly to the assistant.

Step 6

Verify Telegram is live

Check OpenClaw status and confirm Telegram shows as active and healthy before testing messages.

Step 7

Fix slow or broken replies if needed

If Telegram feels slow or messages stop arriving, the most common beginner issue is a duplicate bot token or another process polling the same bot. Replace the old token if needed, then restart and test again.

Step 8

Send a confirmation test

Send yourself a simple confirmation message to make sure delivery works end to end.

Step 9

Use voice replies

Once voice is enabled in your OpenClaw runtime, Telegram can deliver spoken replies as audio. If text works but voice does not, verify your TTS provider is enabled and then test a short spoken reply first.

Simple beginner checklist

  1. Create bot in BotFather
  2. Add token to OpenClaw
  3. Enable Telegram
  4. Restart gateway
  5. Approve DM pairing
  6. Confirm Telegram status is OK
  7. Send a test message
  8. Test a voice reply

Important note about voice

The exact original voice setup sequence was not fully documented during the first pass. What is confirmed is that voice/TTS is currently enabled in the runtime.
