Set Up Telegram + Voice Replies in OpenClaw

A complete beginner-friendly walkthrough. By the end, your assistant will respond in Telegram with natural-sounding voice messages powered by ElevenLabs.

What You Need

  • OpenClaw installed and running on your machine
  • A Telegram account
  • An ElevenLabs account (free tier works — elevenlabs.io)
  • Access to @BotFather inside Telegram
  • Your OpenClaw config file (~/.openclaw/openclaw.json)

Part 1 — Telegram Bot Setup

Step 1 — Create your Telegram bot

Use @BotFather to get a bot token

  1. Open Telegram and search for @BotFather
  2. Send the command /newbot
  3. Follow the prompts — pick a display name and a username (must end in bot)
  4. BotFather will give you a bot token — it looks like 7123456789:AAH...
  5. Save this token. You need it in Step 3.
Treat your bot token like a password. Anyone with it can control your bot.
Step 2 — Open your OpenClaw config

Locate the config file

Your config lives at:

~/.openclaw/openclaw.json

Open it in any text editor. If the file is empty or new, that's fine — you'll build it out below.

Step 3 — Add Telegram to your config

Enable the Telegram channel

Add or update the channels.telegram block in your config:

{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "YOUR_BOT_TOKEN_HERE",
      "dmPolicy": "pairing"
    }
  }
}
What dmPolicy: "pairing" means: When someone DMs your bot, OpenClaw holds the message until you approve that user. This keeps strangers from talking to your assistant.
Step 4 — Restart OpenClaw

Load the new Telegram channel

After saving your config, restart the gateway:

openclaw gateway restart

Then check that Telegram came up clean:

openclaw status

You should see Telegram listed as active/OK.

Step 5 — Start a DM and approve pairing

Connect your Telegram account

  1. Open Telegram and find the bot you just created
  2. Send it any message (like hello)
  3. OpenClaw will log a pairing request with a code
  4. Approve it:
openclaw devices approve YOUR_PAIRING_CODE

Once approved, your Telegram DM is live with the assistant.

Step 6 — Send a test message

Verify two-way communication

Send your bot a message from Telegram. You should get a text reply back within a few seconds.

If replies are slow or never arrive, the most common cause is a duplicate bot token — another process polling the same bot. Fix: generate a fresh token from BotFather with /revoke, update your config, and restart.

Part 2 — ElevenLabs Voice Setup

Step 7 — Create an ElevenLabs account

Sign up and get your API key

  1. Go to elevenlabs.io and create a free account
  2. Once logged in, go to your Profile → API Keys
  3. Click "Create API Key" and copy it
  4. Save it — you need it in Step 9
The free tier gives you 10,000 characters per month. That's plenty for testing. Paid plans unlock more characters, voice cloning, and higher quality models.
Step 8 — Pick a voice

Choose or create a voice for your assistant

  1. In the ElevenLabs dashboard, go to Voice Library
  2. Browse community voices or use a default one
  3. Click on a voice you like — note the Voice ID from the URL or the "..." menu
  4. Example default voice IDs:
    • EXAVITQu4vr4xnSDxMaL — Bella (warm, natural)
    • 21m00Tcm4TlvDq8ikWAM — Rachel (calm, clear)
    • AZnzlk1XvdvUeBnXmlld — Domi (energetic)
You can also create a custom voice clone under Voice Lab → Instant Voice Cloning. Upload a short audio clip and ElevenLabs builds a voice from it.
Step 9 — Set your API key as an environment variable

Make the key available to OpenClaw

Add this line to your shell profile (~/.zshrc or ~/.bashrc):

export ELEVENLABS_API_KEY="your_api_key_here"

Then reload your shell:

source ~/.zshrc

OpenClaw will automatically pick up this environment variable.

Restart the gateway after setting the env variable so OpenClaw can see it: openclaw gateway restart
Step 10 — Add TTS config to OpenClaw

Enable voice replies

Add the messages.tts block to your openclaw.json. Here's a complete working config with both Telegram and ElevenLabs:

{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "YOUR_BOT_TOKEN_HERE",
      "dmPolicy": "pairing"
    }
  },
  "messages": {
    "tts": {
      "auto": "always",
      "provider": "elevenlabs",
      "providers": {
        "elevenlabs": {
          "apiKey": "${ELEVENLABS_API_KEY}",
          "model": "eleven_multilingual_v2",
          "voiceId": "EXAVITQu4vr4xnSDxMaL",
          "voiceSettings": {
            "stability": 0.5,
            "similarityBoost": 0.75,
            "style": 0.0,
            "useSpeakerBoost": true,
            "speed": 1.0
          },
          "applyTextNormalization": "auto",
          "languageCode": "en"
        }
      }
    }
  }
}
Key fields explained:
  • auto: "always" — every reply becomes a voice message. Change to "inbound" to only reply with voice when the user sends voice first.
  • provider: "elevenlabs" — tells OpenClaw to use ElevenLabs for TTS.
  • apiKey: "${ELEVENLABS_API_KEY}" — reads from your environment variable.
  • model: "eleven_multilingual_v2" — best quality model. Use "eleven_v3" for the newest model if available on your plan.
  • voiceId — the ElevenLabs voice ID you picked in Step 8.
  • voiceSettings — fine-tune how the voice sounds:
    • stability (0–1): higher = more consistent, lower = more expressive
    • similarityBoost (0–1): higher = closer to original voice
    • style (0–1): higher = more expressive delivery
    • speed (0.5–2.0): 1.0 = normal pace
Step 11 — Restart and test voice

Hear your assistant speak

  1. Restart the gateway: openclaw gateway restart
  2. Send your bot a message in Telegram
  3. The reply should arrive as a voice message (audio)

You can also test voice directly without waiting for a chat reply:

/tts audio Hello, this is a test of my ElevenLabs voice.

This sends a one-off audio message through Telegram.

Part 3 — Fine-Tuning & Extras

Optional — Voice-only on voice inbound

Only reply with voice when the user sends voice

If you don't want every reply to be audio, change auto to "inbound":

"auto": "inbound"

Now the assistant only sends voice back when you send a voice message first. Regular text messages get regular text replies.

Optional — Use /tts commands

Control voice on the fly

  • /tts status — see current TTS settings, provider, and voice
  • /tts on — enable auto voice replies
  • /tts off — disable auto voice replies
  • /tts audio Hello world — send a one-off voice message
  • /tts provider elevenlabs — switch provider mid-session
Optional — Add a voice persona

Give your assistant a consistent spoken identity

You can define a persona with a name and style that carries across sessions:

"messages": {
  "tts": {
    "auto": "always",
    "provider": "elevenlabs",
    "persona": "assistant",
    "personas": {
      "assistant": {
        "label": "InnerG Assistant",
        "description": "Warm, confident, efficient.",
        "provider": "elevenlabs",
        "providers": {
          "elevenlabs": {
            "voiceId": "EXAVITQu4vr4xnSDxMaL",
            "modelId": "eleven_multilingual_v2",
            "seed": 42,
            "voiceSettings": {
              "stability": 0.55,
              "similarityBoost": 0.8,
              "style": 0.15,
              "useSpeakerBoost": true,
              "speed": 1.05
            }
          }
        }
      }
    },
    "providers": {
      "elevenlabs": {
        "apiKey": "${ELEVENLABS_API_KEY}",
        "model": "eleven_multilingual_v2"
      }
    }
  }
}
seed makes the voice more deterministic — same text produces more consistent output each time. Value is any integer from 0 to 4,294,967,295.

Troubleshooting

Problem — Telegram replies are slow or missing

Fix: Check for duplicate bot tokens

  • Only one process should poll your bot token at a time
  • If you set up the bot somewhere else before, revoke and regenerate the token in BotFather with /revoke
  • Update the token in your config and restart
Problem — No voice / only text replies

Fix: Check TTS config and API key

  1. Run /tts status in Telegram to see if TTS is active
  2. Confirm ELEVENLABS_API_KEY is set: echo $ELEVENLABS_API_KEY
  3. Check that messages.tts.auto is set to "always" or "inbound"
  4. Check your ElevenLabs dashboard — did you hit your character limit?
  5. Restart the gateway after any config or env change
Problem — Voice sounds robotic or wrong

Fix: Tune voiceSettings

  • Increase stability (try 0.6–0.75) for more consistent delivery
  • Increase similarityBoost (try 0.8–0.9) to sound closer to the original voice sample
  • Try a different voice ID — some voices respond better to certain settings
  • Switch to "eleven_v3" model if available — it's more natural-sounding
Problem — Pairing code not showing up

Fix: Check OpenClaw logs

  • Run openclaw status to see if Telegram is online
  • Check the gateway logs for errors related to the Telegram channel
  • Make sure the bot token is correct and hasn't been revoked
